# Implementation Plan: Questions CSV Import

**Branch**: `004-questions-csv-import` | **Date**: 2026-05-03 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/004-questions-csv-import/spec.md`

## Summary

Implement bulk CSV import for the question bank via two new HTTP endpoints (`POST /api/v1/questions/import` and `GET /api/v1/questions/import/template`) living entirely within `internal/question/`. The import is synchronous, validates each row against existing categories, and supports two error modes: `abort` (all-or-nothing, single DB transaction) and `skip` (best-effort, row-by-row inserts). No new migrations or packages are required.

## Technical Context

**Language/Version**: Go 1.25  
**Primary Dependencies**: gofiber/fiber v2, jackc/pgx v5, encoding/csv (stdlib), google/uuid v1  
**Storage**: PostgreSQL — `questions` and `categories` tables (migration 001, no new migration)  
**Testing**: Manual smoke test per `quickstart.md`  
**Target Platform**: Linux server (Docker Compose)  
**Project Type**: web-service (backend)  
**Performance Goals**: 500-row CSV import completes within 10 seconds  
**Constraints**: max 500 data rows per import, max 5 MB file size  
**Scale/Scope**: Single-admin MVP

## Phase 0: Research

Research complete. See [research.md](research.md).

Key decisions:
1. **CSV parsing**: `encoding/csv` stdlib — no new dependencies
2. **Abort mode**: Validate all rows first → open single pgx transaction → batch INSERT → commit/rollback
3. **Skip mode**: Reuse `repo.Create()` per valid row; no transaction
4. **Category lookup**: Load full category map (`lower(name) → uuid`) once before row processing
5. **Duplicate detection**: Per-row `EXISTS` query; soft warning; does not block import
6. **File size**: Explicit check in handler (`header.Size > 5*1024*1024`) + Fiber `BodyLimit` config update
7. **Row limit**: Count rows after CSV parse; reject if > 500 before any DB access

## Phase 1: Design

### Data Model

See [data-model.md](data-model.md). No new DB tables. New Go types:
- `ImportRow` — parsed CSV row with 1-based row number
- `ImportResult` — `{Imported int, Skipped int, Errors []RowError}`
- `RowError` — `{Row int, Message string}` (JSON: `row`, `message`)
- Sentinel: `ErrInvalidOnError`

### HTTP Contracts

See [contracts/http.md](contracts/http.md).

- `GET /api/v1/questions/import/template` — public, returns raw CSV with headers only
- `POST /api/v1/questions/import` — admin JWT required, multipart/form-data

### Repository additions

```
FindCategoryMap(ctx context.Context) (map[string]uuid.UUID, error)
  → SELECT id, LOWER(name) FROM categories WHERE deleted_at IS NULL
  → returns map[lower_name]→uuid

CheckDuplicate(ctx context.Context, text string, categoryID uuid.UUID) (bool, error)
  → SELECT EXISTS(SELECT 1 FROM questions WHERE LOWER(text)=LOWER($1) AND category_id=$2 AND deleted_at IS NULL)

CreateBatch(ctx context.Context, tx pgx.Tx, questions []Question) error
  → executes N INSERT statements within the supplied transaction
  → used by abort mode only
```

### Service additions

```
Import(ctx context.Context, r io.Reader, onError string) (ImportResult, error)
  → parses CSV
  → validates on_error value
  → counts rows; rejects if > 500
  → loads category map (FindCategoryMap)
  → validates each row; runs duplicate check on valid rows
  → abort mode: if any hard errors → return result (no DB write)
                 else → CreateBatch in transaction → return result
  → skip mode: for each row → Create or record error → return result
```

### Handler additions

```
ImportTemplate(c *fiber.Ctx) error
  → sets Content-Type: text/csv
  → sets Content-Disposition: attachment; filename="questions_template.csv"
  → writes header row string to response

Import(c *fiber.Ctx) error
  → validates on_error field (default "abort")
  → reads file from multipart field "file"
  → checks file size
  → calls svc.Import(ctx, file, onError)
  → wraps result in api.DataResponse envelope
```

### Route registration in main.go

```go
// Public route (no auth)
v1.Get("/questions/import/template", questionHandler.ImportTemplate)

// Protected route (admin JWT)
protected.Post("/questions/import", questionHandler.Import)
```

**Important**: The template route must be registered on `v1` (not `protected`) to skip the `RequireAdmin` middleware. The import route goes on `protected`.
